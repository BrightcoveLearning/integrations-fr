Connecteur WordPress-Brightcove : Foire aux questions

Cette rubrique contient les questions fréquemment posées sur Wordpress-Brightcove Connector.

Puis-je exécuter le plugin sur une installation WordPress qui n'est pas accessible publiquement ?

Oui, cela fonctionnera, qu'il soit public ou non, mais les fonctionnalités nécessitant un rappel sur votre site telles que les mises à jour d'état, le téléchargement de vidéos, etc. ne seront pas disponibles.

 Existe-t-il des filtres pour les développeurs de plugin/thèmes ? 

brightcove_account_actions = [modifier, supprimer] ;//Quelles actions sont disponibles lors de la manipulation d'une source Brightcove.

Est-ce que cela fonctionnera sur multisite ?

Oui, ça le fera.

Puis-je utiliser plus d'un compte Brightcove ?

Oui, vous pouvez ajouter des sources à partir de nombreux comptes Brightcove si vous le souhaitez.

Puis-je définir des autorisations différentes pour les utilisateurs ?

Oui. Vous trouverez une matrice des autorisations utilisateur dans ce document. Pour modifier les rôles des utilisateurs, nous vous recommandons d'utiliser la fonction add_cap WordPress : https://codex.wordpress.org/add_cap

Comment fonctionne la synchronisation ?

Le plugin extrait simplement les informations directement depuis l'API Brightcove pour l'affichage et ne synchronise pas les vidéos localement.

Comment puis-je augmenter la taille maximale du fichier de téléchargement ?

La taille maximale du fichier est déterminée par votre serveur web et la configuration PHP. Vous devez définir la valeur upload_max_filesize et post_max_size dans votre fichier php.ini. php_ini_loaded_file () peut vous aider à trouver où se trouve votre PHP.ini.

Comment activer le suivi des modifications pour voir qui a mis à jour une vidéo et quand il l'a fait ?

Créez un champ personnalisé de type « texte' avec le nom interne « _change_history ». Chaque fois qu'une vidéo est mise à jour, le nom d'utilisateur et l'heure actuelle sont ajoutés à la liste des modifications enregistrées dans ce champ.
