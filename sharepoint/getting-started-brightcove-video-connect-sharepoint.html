    Installation de Brightcove Video Connect pour SharePoint

    Avant de commencer l'installation, consultez les informations suivantes sur les conditions préalables et les autorisations :

    • Avant qu'un utilisateur puisse ajouter une application pour SharePoint, un membre du groupe Administrateurs de batterie doit configurer l'environnement pour prendre en charge les applications pour SharePoint. Pour plus d'informations, consultez Configurer un environnement pour les applications pour SharePoint 2013 ou SharePoint 2016 ou SharePoint 2019.
    • Un utilisateur doit disposer des autorisations Gérer le site Web et Créer des sous-sites pour ajouter une application pour SharePoint. Par défaut, ces autorisations sont disponibles uniquement pour les utilisateurs disposant du niveau d'autorisation Contrôle total ou qui font partie du groupe Propriétaire du site.
    • Lorsqu'un utilisateur ajoute une application pour SharePoint, l'application demande les autorisations dont il a besoin pour fonctionner (par exemple, l'accès à la recherche ou pour créer une liste). Les utilisateurs qui ne disposent pas de ces autorisations sont informés qu'ils ne disposent pas des autorisations suffisantes et que l'application ne peut pas être ajoutée. L'utilisateur peut contacter un administrateur de site ou de batterie de serveurs pour voir si l'administrateur peut ajouter l'application.
    • Un utilisateur connecté à un site en tant que compte système ne peut pas installer une application. Le compte système ne peut pas importer de licences d'application car cela peut entraîner des problèmes de performances.

    Ajout de l'application Brightcove Video Connect au catalogue d'applications

    Configuration du catalogue d'applications

    Pour installer le connecteur Brightcove, la première étape consiste à configurer le catalogue d'applications dans SharePoint. Utilisez les références ci-dessous pour terminer cette étape.

    • Gérer le catalogue d'applications dans SharePoint 2013
    • Gérer le catalogue d'applications dans SharePoint 2016
    • Gérer le catalogue d'applications dans SharePoint 2019

    Ajout de l'application Brightcove Video Connect au catalogue

    Une fois le catalogue d'applications configuré, suivez les étapes ci-dessous :

    1. Vérifiez que le compte d'utilisateur qui exécute cette procédure est membre du groupe Propriétaires de site ou concepteurs du catalogue d'applications.
    2. Sur le site du catalogue d'applications, cliquez sur la liste Applications pour SharePoint .
    3. Dans la page Applications pour SharePoint , cliquez sur nouvel élément.
    4. Dans la zone Choisir un fichier , cliquez sur Parcourir , puis recherchez le dossier contenant le fichier BrightCoveConnectorApp .
    5. Sélectionnez l'application, puis cliquez sur Ouvrir.
    6. Cliquez sur OK pour télécharger l'application.
    7. Dans la zone Détails de l'article, indiquez le nom, le titre, la description courte, l'URL de l'icône et d'autres paramètres de l'application. Assurez-vous que la case à cocher Activé est activée pour que les utilisateurs puissent voir l'application sur leurs sites. Vous pouvez activer la case à cocher En vedette pour répertorier l'application dans la vue Contenu en vedette du catalogue d'applications.
    8. Cliquez sur Save.

    Installation de l'application Brightcove Video Connect dans un site

    Suivez les étapes ci-dessous pour installer le connecteur Brightcove dans un site :

    1. Vérifiez que le compte d'utilisateur qui exécute cette procédure est membre du groupe Propriétaires de site.
    2. L'application peut être ajoutée à un site de l'une des deux manières suivantes :
      1. Sur n'importe quelle page, dans le menu Paramètres , sélectionnez Ajouter une option d'application .
      2. Dans Contenu du site à l'aide de l'option Ajouter une application .
    3. Ensuite, cliquez sur « De votre organisation », puis sur l'application BrightCoveConnector .
    4. Dans la boîte de dialogue Accorder l'autorisation, cliquez sur « Faire confiance ».
    5. À ce stade, l'application BrightCoveConnector pour SharePoint est ajoutée et apparaît dans la section Applications de votre liste Contenu du site. Il est maintenant disponible pour une utilisation sur le Site.

    Avant d'utiliser le connecteur Brightcove, certaines listes requises doivent être provisionnées en procédant comme suit :

    • Lorsque l'application est instanciée pour la première fois, ou si les listes ne sont pas présentes sur le Site local, l'utilisateur recevra jusqu'à 3 options pour permettre le provisionnement des listes à l'intérieur du Site.

    • Lorsque vous cliquez dessus, chaque bouton affiche un message à l'effet :
    • Lorsque vous cliquez sur « Créer une liste de paramètres », en plus de créer la liste des paramètres Brightcove, une invite de modification de cette liste s'affiche pour s'assurer que l'URL proxy requise pour la prise en charge de l'API Brightcove est configurée.
    • Cliquez sur ce lien pour ouvrir la liste des paramètres de Brightcove pour modifier immédiatement les informations d'accès au serveur proxy. Le titre doit être défini sur « Proxy ». ProxyURL est spécifique à l'emplacement du serveur proxy que vous utiliserez pour accéder à l'API Brightcove. Pour plus de détails sur l'utilisation du serveur proxy, reportez-vous à la section Utilisation du serveur proxy pour la communication de l'API Brightcove .
    • La mise en service de liste créera les listes suivantes dans le site :
      • Comptes : liste personnalisée représentant les objets Compte.
      • Vidéos - Une liste personnalisée représentant les objets vidéo.
      • Paramètres - Liste pour stocker les détails du serveur proxy.

    Installation du package de solution Player pour SharePoint Online

    Le package de solutions Brightcove Player inclus dans le package Brightcove Video Connect permet à l'utilisateur d'intégrer des lecteurs vidéo Brightcove sur les pages du site SharePoint. Le package de solutions Brightcove Player utilise des contrôles SharePoint disponibles qui sont compatibles avec SharePoint Online et SharePoint Server et peuvent donc être utilisés avec les deux. Une solution sandbox Player WebPart pour l'intégration de vidéos est également disponible dans le package Brightcove Video Connect, mais ne peut être utilisée qu'avec SharePoint Server sur site car SharePoint Online et SharePoint Server hébergés par Microsoft ont des restrictions de sécurité interdisant les solutions sandbox ; cette solution est décrit dans la section suivante.

    Pour intégrer un lecteur Brightcove dans SharePoint Online, le package de solution décrit ici permet à l'utilisateur de choisir un lecteur vidéo ou une liste de lecture, puis le code d'intégration du lecteur est généré pour permettre à l'utilisateur de copier/coller dans la boîte de dialogue Code d'intégration SharePoint.

    REMARQUE - Cette section nécessite le fichier de solution BrightCovePlayer.wsp.

    1. Dans le site racine de la collection de sites, où l'application Brightcove est installée, accédez à Paramètres du > site - Administrations de collection de sites - Sécurité des champs > HTML.
    2. Ajouter players.brightcove.net en tant que domaine iframe autorisé
    3. Dans le site racine de la collection Site, où l'application Brightcove est installée, accédez à Paramètres du site - Galeries > Web Designer - > Solutions.
    4. Téléchargez le fichier « BrightCovePlayer.wsp » dans la Galerie de solutions et activez le fichier.
    5. Accédez à Paramètres du site - Actions > du site - > Gérer les fonctionnalités du site.
    6. Activez la fonctionnalité « Brightcove Player ».
    7. Vérifiez que la bibliothèque de documents BrightCoveConnector Resources a été créée lors de l'installation.
    8. Si ce n'est pas le cas, créez une bibliothèque de documents appelée BrightCoveConnector Resources
    9. et téléchargez BrightcovePlayer.css et BrightcovePlayer.js dans cette bibliothèque de documents.

    Installation du serveur proxy pour la communication de l'API Brightcove

    Étant donné que le Connector est une application JavaScript côté client, un serveur proxy est requis pour courtier les communications authentifiées vers l'API Brightcove. Vous trouverez plus d'informations dans le document API Brightcove.

    Une application proxy .net est fournie dans le package Brightcove Video Connect. Le serveur proxy doit être déployé dans un emplacement accessible sur le Web, tel que sur Microsoft Azure. Pour des raisons de sécurité, le proxy est conçu pour autoriser uniquement les connexions à partir de sites Web inclus dans le fichier web.config installé avec le proxy. La clé brightcove:origines du fichier web.config doit inclure l'URL du site Web SharePoint pour l'application BrightCoveConnector ainsi que l'URL du site sur lequel le composant BrightCovePlayer s'exécutera. Voir l'exemple ci-dessous :

    <add key="brightcove:origins" value="https://brightcovecms-a3da847a1c9e33.sharepoint.com, https://brightcovecms.sharepoint.com" />

    Plusieurs sites peuvent être répertoriés sous la forme d'une chaîne séparée par des virgules.

    Lancement de Brightcove Video Connect pour SharePoint

    Pour lancer l'application, l'utilisateur peut cliquer sur l'application BrightCoveConnector dans la liste Contenu du site, ou cliquer sur un lien de menu si cela est configuré par l'administrateur SharePoint.

    Dans les deux cas, l'utilisateur sera amené à la page d'accueil de l'application Connector. Cette page pourrait être configurée par le client pour inclure des messages supplémentaires spécifiques à l'entreprise, des liens supplémentaires, etc. Comme décrit dans la section d'installation, selon l'appartenance au groupe de l'utilisateur, il peut ou non afficher les onglets Gestion de compte et Ajouter des vidéos.

    La navigation principale dans la colonne de gauche du Connecteur est toujours présente et permet à l'utilisateur de naviguer dans toutes les sections auxquelles il a accès (voir la section ci-dessus concernant les autorisations). Voici une brève description de l'objectif de chaque composant du connecteur :

    • Accueil - La page de destination lors de la première entrée dans le connecteur.
    • Gestion des comptes - Fournit l'interface permettant de saisir les informations de compte Brightcove Video Cloud dans SharePoint.
    • Ajouter des vidéos - Téléchargez de nouvelles vidéos dans Video Cloud. Un objet vidéo est également créé dans SharePoint (à l'exception du fichier vidéo réel).
    • Gérer les vidéos - Parcourez et modifiez des vidéos existantes, ainsi que des vidéos depuis Video Cloud dans SharePoint.
    • Gérer les playlists - Créez de nouvelles playlists ainsi que parcourez et gérez les playlists existantes.

    Il existe également des composants Ajouter une vidéo à partir de Brightcove et Modifier la vidéo qui seront affichés dans la navigation principale lors de l'exécution de ces opérations.

    Liaison aux comptes Video Cloud

    La section Gestion des comptes de Connector permet d'entrer les informations relatives à chacun des comptes Brightcove Video Cloud liés qui seront accessibles sur le Site. Les comptes Brightcove ne peuvent pas être créés à partir de SharePoint, ils doivent d'abord être créés dans Video Cloud. Les informations du compte Brightcove doivent être saisies dans SharePoint telles qu'elles sont utilisées lors de l'accès à l'API Brightcove pour communiquer avec Video Cloud. Il fournit également les moyens d'attribuer des autorisations d'accès spécifiques à un compte. Pour les clients d'entreprise multi-comptes, il permet de contrôler quels utilisateurs peuvent accéder à chaque compte et s'ils peuvent simplement parcourir ou télécharger des vidéos.

    Page de destination Gestion des comptes

    La page d'accueil Gestion des comptes affiche la liste actuelle des comptes configurés, ainsi que les groupes affectés aux auteurs et aux visionneuses.

    1. Titre du compte - Comptes actuellement configurés (ou au moins créés).
    2. AuthorsGroup - Le groupe SharePoint a attribué les droits « Auteur » pour le compte.
    3. ViewersGroup - Le groupe SharePoint a attribué les droits « Viewer » pour le compte.
    4. Ajouter un nouveau compte - Ouvrez l'écran de création de compte pour configurer un nouveau compte.

    Création/modification d'un compte

    Qu'il s'agisse de créer un compte ou de modifier un compte existant, le processus et les options sont les mêmes. Toutes les propriétés du compte (à l'exception des groupes Auteur et Viewers) doivent d'abord être obtenues à partir du compte Video Cloud. Voici les propriétés requises pour chaque compte.

    1. Nom du compte : nom du compte. Ce nom n'est utilisé que dans SharePoint, il n'est donc pas nécessaire d'être le même que dans Video Cloud.
    2. DefaultVideoPlayerID - Définit le lecteur par défaut pour les vidéos.
    3. DefaultPlayListPlayerID - Définit le lecteur de liste de lecture par défaut.
    4. ID de compte : ID de compte Brightcove de Video Cloud.
    5. ClientID - ID client d'authentification de l'API Brightcove.
    6. Client Secret - Secret client d'authentification de l'API Brightcove.
    7. Emplacement de stockage temporaire - Emplacement de stockage temporaire à utiliser lors du téléchargement de vidéos et de pistes de texte vers Video Cloud. Le menu déroulant permet de sélectionner Amazon Web Services (AWS) ou Dropbox.
    8. ID de clé d'accès AWS - Si vous utilisez AWS, l'ID de clé d'accès est entré ici.
    9. Clé d'accès secrète AWS - Si vous utilisez AWS, la clé d'accès secrète est entrée ici.
    10. Nom du compartiment AWS - Si vous utilisez AWS, le nom du compartiment AWS est entré ici.
    11. ID de compte Dropbox - Si vous utilisez Dropbox, l'ID de compte Dropbox est entré ici.
    12. Token d'accès Dropbox - Si vous utilisez Dropbox, le jeton d'accès Dropbox est entré ici.
    13. Champs personnalisés : permet de mapper les champs personnalisés créés dans Video Cloud pour ce compte. Voir la section Mappage de champs personnalisés pour plus d'informations.
    14. Groupe d'auteurs, Groupe de visionneuses - Groupes de définition d'autorisations côté SharePoint pour le compte.
      1. Les membres du groupe d'auteurs (dans ce cas, le groupe « Propriétaires » SharePoint) ont la possibilité d'exécuter des fonctions dans le compte qui nécessitent un jeton d'écriture telles que l'ajout d'une vidéo, l'édition et la mise à jour d'une vidéo existante, la création/modification d'une playlist, etc. Ils peuvent également effectuer toutes les fonctions de type lecture telles que la navigation et la recherche de vidéos et de listes de lecture.
      2. Les membres du groupe de visionneuses (dans ce cas, le groupe « Visiteurs » SharePoint) ne sont autorisés que pour les fonctions de type lecture telles que la navigation et la recherche de vidéos et de playlists.

    Pour modifier l'une des propriétés de la liste de comptes ci-dessus, l'utilisateur peut soit cliquer sur le nom de la propriété ou sa valeur (le cas échéant) pour ouvrir la boîte de dialogue contextuelle de propriété :

    Saisissez les informations d'identification du compte de Brightcove dans SharePoint

    Pour que SharePoint puisse se connecter à Brightcove Video Cloud, les informations d'identification du compte de Brightcove doivent être ajoutées à SharePoint. Cette section décrit comment localiser les informations d'identification du compte dans Brightcove. La section suivante décrit comment ajouter ces informations d'identification au connecteur SharePoint.

    1. Connectez-vous à Brightcove Video Cloud. Notez la valeur de l'ID de l'éditeur.
    2. Accédez à Admin.
    3. Accédez à l'authentification API.
    4. Enregistrez une nouvelle demande.
    5. Nommez l'application, définissez l'accès au compte et activez les API.
    6. Notez l'ID client et le secret client. Vous devrez les entrer dans SharePoint dans la section suivante.

    Pour définir l'ID de compte Brightcove, cliquez sur le bouton « ID de compte » et saisissez l'ID d'éditeur Brightcove que vous avez enregistré à l'étape précédente sur le mode suivant :

    Pour définir l'ID client Brightcove, sélectionnez le bouton « ID client » et entrez l'ID client que vous avez enregistré à l'étape précédente sur le modal suivant :

    Pour définir le secret client Brightcove, cliquez sur le bouton « Client Secret » et saisissez le secret client que vous avez enregistré à l'étape précédente sur le modal suivant :

    1. Valeur - Saisissez la valeur de la propriété. La valeur peut être saisie à l'aide du coupe-coller à partir des données Video Cloud, si disponible.
    2. Fermer - Fermez la boîte de dialogue des propriétés sans apporter de modifications.
    3. OK - Enregistrez la valeur dans la liste des propriétés et fermez la boîte de dialogue.

    Définir des lecteurs Video Cloud

    Pour définir l'ID de lecteur qui doit être utilisé comme lecteur vidéo unique par défaut dans SharePoint, sélectionnez le bouton « DefaultVideoPlayerID », entrez l'ID de lecteur de Brightcove sur le mode suivant :

    1. Valeur - Saisissez la valeur de la propriété. La valeur peut être saisie à l'aide du coupe-coller à partir des données Video Cloud, si disponible.
    2. Fermer - Fermez la boîte de dialogue des propriétés sans apporter de modifications.
    3. OK - Enregistrez la valeur dans la liste des propriétés et fermez la boîte de dialogue.

    Pour définir l'ID de lecteur qui doit être utilisé comme lecteur vidéo de playlist par défaut dans SharePoint, cliquez sur le bouton « DefaultPlaylistPlayerID » et entrez l'ID du lecteur dans Brightcove.

    Définir des champs personnalisés

    Lors de la configuration d'un compte dans le connecteur Brightcove, les champs personnalisés créés dans Video Cloud peuvent être ajoutés au profil de compte dans le Connecteur afin qu'ils apparaissent dans les zones Ajouter des vidéos et Gérer les vidéos, et que leurs valeurs soient stockées dans SharePoint pour la recherche. Lorsque les champs personnalisés sont mappés, ils sont répertoriés dans la section Champs personnalisés de l'écran Gestion des comptes, et indiqueront s'ils sont obligatoires ou non :

    Pour ajouter, supprimer ou modifier des champs personnalisés, cliquez sur le bouton « Modifier les champs personnalisés ».     Pour ajouter une nouvelle playlist à un compte, l'utilisateur clique sur le bouton « Ajouter une nouvelle playlist ». Le formulaire Ajouter une liste de lecture s'ouvre :

     

    Notez que les vidéos peuvent être déplacées vers le haut et vers le bas dans la liste de lecture en sélectionnant la vidéo à déplacer et en activant l'option « Déplacer la vidéo » pour utiliser les boutons Haut/Bas. Plusieurs vidéos peuvent être déplacées à la fois.

    1. Nom - Nom de la nouvelle liste de lecture.
    2. Description - Une brève description de la liste de lecture.
    3. Type - Type de liste de lecture - Manuel ou Smart. Cet exemple montre une liste de lecture manuelle. L'exemple de la section « Modification d'une liste de lecture » montre les options d'une liste de lecture intelligente.
    4. Vidéos disponibles - Dans le cas d'une liste de lecture manuelle, affichez la zone de recherche et la liste des vidéos disponibles dans le compte. Cela permet à l'utilisateur de restreindre les choix vidéo à ajouter à la liste de lecture.
    5. Liste des vidéos disponibles - Répertorier les vidéos disponibles dans le compte ou filtre en fonction du terme recherché. Une ou plusieurs vidéos peuvent être sélectionnées en cliquant sur les vidéos. Les vidéos sélectionnées sont mises en surbrillance en vert. Pour désélectionner une vidéo, l'utilisateur clique à nouveau dessus. Cela permet de déplacer plusieurs vidéos dans la liste de lecture en une seule action.
    6. Flèches d'action - Déplace les vidéos sélectionnées dans et hors de la liste de lecture.
    7. Vidéos de cette liste de lecture - Les vidéos actuelles de la liste de lecture sont affichées ici.
    8. Enregistrer - Engagez les modifications et commencez le processus de création de la nouvelle liste de lecture dans Video Cloud. REMARQUE - Les listes de lecture ne sont pas enregistrées dans SharePoint.

    Modification d'une liste de lecture existante

    La modification d'une liste de lecture existante est commencée en cliquant soit sur le nom, soit sur le lien « Modifier » dans la liste des listes de lecture. Cela ouvre l'affichage suivant pour modifier les options de la liste de lecture. Cet exemple montre comment afficher les options d'une liste de lecture intelligente. Lors de la modification de l'un des champs, à l'exception de l'ID de la liste de lecture et du compte, peut être modifié :

    1. Nom - Nom de la liste de lecture.
    2. Description - Une brève description de la liste de lecture.
    3. Type - Type de liste de lecture - Manuel ou Smart. Cet exemple montre comment couvrir les options d'une liste de lecture intelligente.
    4. Tags - Dans le cas d'une liste de lecture intelligente, ce champ permet à l'utilisateur de spécifier les critères et les balises à utiliser pour identifier les vidéos à inclure dynamiquement dans la liste de lecture.
      • La liste déroulante Critères comprend « Contenir un ou plusieurs » ou « Contenir tout ».
    5. Ordre - Cela permet à l'utilisateur de spécifier comment l'ordre des vidéos correspondantes est défini. Les sélections comprennent les éléments suivants :
      • Alphabétique
      • Date d'activation (la plus récente en premier)
      • Date d'activation (la plus ancienne en premier)
      • Nombre total de lectures
      • Nombre de lectures sur les sept derniers jours
      • Date de début (la plus ancienne en premier)
      • Date de début (la plus récente en premier)
    6. Limiter à - Définit le nombre maximal de vidéos qui seront incluses dans la liste de lecture en fonction des critères sélectionnés.
    7. Aperçu - Répertorier les vidéos correspondant aux critères et les afficher dans la fenêtre « Vidéos dans cette playlist ».
    8. Enregistrer - Engagez les modifications et commencez le processus de mise à jour de la liste de lecture dans Video Cloud. REMARQUE - Les listes de lecture ne sont pas enregistrées dans SharePoint.
    9. Supprimer la liste de lecture - Cette option marquera la liste de lecture comme supprimée dans Video Cloud. La liste de lecture n'apparaîtra plus dans la liste des listes de lecture Connector.

    Ajout de vidéo à un site SharePoint

    Le connecteur SharePoint permet aux auteurs de contenu d'insérer des médias dans leurs Sites de différentes manières. Ce chapitre décrit les options disponibles.

    Part WebPart Brightcove Player pour SharePoint sur site

    La solution sandbox de composants WebPart Brightcove Player génère un composant WebPart placé sur une page du site Web interne ou externe et lit la vidéo Brightcove. SharePoint Online limite la possibilité pour les utilisateurs d'ajouter des composants WebPart personnalisés de sorte que ce mécanisme d'insertion de vidéos n'est disponible que pour les installations SharePoint sur site.

    Le composant WebPart a également été complété avec certaines des mêmes fonctionnalités dans la console de gestion, y compris la sélection de compte et la recherche avancée avec tri et commande pour trouver des vidéos et des listes de lecture plus rapidement.

    Pour afficher les options du composant WebPart Player, la page SharePoint doit être en mode « Modifier » (1), ce qui expose l'option permettant de modifier le composant WebPart (2). Les options spécifiques au composant WebPart se trouvent dans la section « Configuration de Brightcove » (3) de l'éditeur de composants WebPart.

    L'extension de la section Configuration de Brightcove révèle les fonctionnalités mises à jour de la solution sandbox Player WebPart.

    Sélection d'une vidéo à lire dans le composant WebPart

    1. Configuration de Brightcove - Cette section de l'éditeur de composants WebPart permet d'accéder aux paramètres de configuration du composant WebPart et de sélectionner les vidéos/listes de lecture à lire.
    2. Largeur du joueur, Hauteur du joueur - Définissez les dimensions du joueur.
    3. ID du joueur - Indiquez l'ID du joueur à utiliser. Il est important que le lecteur sélectionné soit du même compte que la vidéo sélectionnée s'il s'agit d'un environnement multi-compte. Il n'y a pas de « sélecteur » pour le lecteur, de sorte que l'ID de Video Cloud doit être entré manuellement.
    4. ID vidéo - Le champ ID vidéo est renseigné à l'aide d'un sélecteur vidéo. Si l'utilisateur clique sur la zone de points de suspension située à droite du champ, la fenêtre Sélecteur s'affiche.
    5. Fenêtre Sélecteur
      • Sélectionnez Compte - Affichez les comptes auxquels l'utilisateur actuel est autorisé à accéder.
      • Vidéos disponibles - Filtrer les résultats vidéo à ceux contenant le terme. Le nom, les balises et les informations de description ainsi que les champs de métadonnées SharePoint, s'ils sont définis, sont recherchés. Il n'est possible de rechercher que dans un compte à la fois ; la recherche entre comptes n'est pas prise en charge.
      • Trier par et classer par
      • Fenêtre de listage vidéo - Cliquez sur une vidéo pour la sélectionner pour la lecture. La liste est limitée à l'affichage de 30 vidéos, de sorte que toutes les vidéos du compte ne peuvent pas s'afficher lors de la première sélection d'un compte. L'utilisation de la fonctionnalité de recherche permet d'affiner la liste pour trouver la vidéo souhaitée.
      • Bouton OK - Valider la sélection et renseigner le champ ID vidéo avec la valeur d'ID appropriée.

    Une fois l'ID du lecteur et l'ID vidéo définis, le composant WebPart est configuré et prêt à être joué !

    Sélection d'une liste de lecture à lire dans le composant WebPart

    Le processus de sélection d'une playlist à lire est presque identique à la sélection d'une vidéo.

    1. Configuration de Brightcove - Cette section de l'éditeur de composants WebPart permet d'accéder aux paramètres de configuration du composant WebPart et de sélectionner les vidéos/listes de lecture à lire.
    2. Largeur du joueur, Hauteur du joueur - Définissez les dimensions du joueur.
    3. ID du joueur - Indiquez l'ID du lecteur de lecture à utiliser. Il est important que le lecteur sélectionné soit du même compte que la vidéo sélectionnée s'il s'agit d'un environnement multi-compte et qu'il soit configuré pour afficher une liste de lecture.     4. ID de liste de lecture - L'ID vidéo est renseigné à l'aide d'un sélecteur vidéo. Si l'utilisateur clique sur la zone de points de suspension située à droite du champ, la fenêtre Sélecteur s'affiche.
    5. Fenêtre Sélecteur
      • Sélectionnez Compte - Affichez les comptes auxquels l'utilisateur actuel est autorisé à accéder.
      • Trier par et classer par
      • Fenêtre de liste de liste de lecture - Cliquer sur une liste de lecture la sélectionnera pour la jouer. La liste est limitée à l'affichage de 30 playlists, de sorte que toutes les listes de lecture du compte ne peuvent pas s'afficher lors de la première sélection d'un compte. L'utilisation de la fonctionnalité de recherche permet d'affiner la liste pour trouver la liste de lecture souhaitée.
      • Bouton OK - Valider la sélection et renseigner le champ ID de la liste de lecture avec la valeur d'ID appropriée.

    Une fois l'ID de playlist et l'ID vidéo définis, le composant WebPart est configuré et prêt à être joué !
